# منصة التنظيم المدرسي والموارد التعليمية

منصة ويب عربية باتجاه RTL تجمع روابط المنصات التعليمية، الأدلة والأنظمة، الأدوات المساندة، وركن الدعم المدرسي في واجهة واحدة. تتضمن المنصة تسجيل بيانات المدرسة محليًا، مساعدًا يعتمد على ملفات المعرفة، مكتبة تقارير Word، وصفحات إدارية داخلية لمتابعة المعرفة والأسئلة غير المجابة.

تتكون المنصة من واجهة HTML/CSS/JavaScript تُخدم كـStatic Assets، مع Cloudflare Worker للمسارات الخلفية مثل `/api/chat` وإدارة الأسئلة غير المجابة. لا يستخدم المشروع إطار واجهة أو خطوة build.

## حالة الإصدار

- الإصدار الجاري تجهيزه محليًا: `v1.5.1` — غير منشور.
- الإصدار المنشور الحالي: `v1.5.0`.
- لا يعني تحديث رقم الحزمة أو الواجهة إنشاء Git tag أو نشر الإصدار.

## المتطلبات

- Node.js وnpm لتشغيل الخادم المحلي وأدوات الجودة.
- الاعتماديات المثبتة من `package-lock.json`.
- متصفح Chromium عند تشغيل اختبارات Playwright.
- صلاحيات Cloudflare المناسبة عند النشر فقط؛ لا يحتاج التشغيل المحلي الثابت إلى أسرار.

لتثبيت الاعتماديات المطابقة لملف القفل:

```powershell
npm ci
```

وعند تجهيز Playwright لأول مرة:

```powershell
npx playwright install chromium
```

## التشغيل المحلي

من جذر المشروع:

```powershell
node local-server.cjs
```

ثم افتح:

```text
http://127.0.0.1:4173/index.html
```

الخادم المحلي يقدم الملفات الثابتة فقط. لا يمثل Cloudflare Worker، لذلك لا يمكن الاعتماد عليه لاختبار `/api/chat` أو D1 أو Workers AI أو Vectorize. يجب إبقاء الخادم شغالًا أثناء اختبارات Playwright، ثم إيقافه بعد الانتهاء.

## أوامر الجودة والاختبار

| الأمر | الغرض |
|---|---|
| `npm run check:syntax` | فحص صياغة ملفات JavaScript الأساسية والاختبارات والأدوات |
| `npm run lint:js` | تشغيل ESLint |
| `npm run lint:css` | تشغيل Stylelint |
| `npm run lint:html` | تشغيل HTMLHint |
| `npm run check:links` | فحص الروابط المحلية ومواضع الروابط المؤجلة |
| `npm run check:knowledge` | معاينة ملفات المعرفة والمقاطع دون رفع |
| `npm run test:assistant-security` | اختبارات طلبات المساعد والتعقيم وعدم تسريب debug |
| `npm run test:admin-unanswered` | اختبارات إدارة الأسئلة غير المجابة وpagination |
| `npm run test:e2e` | اختبارات Chromium الوظيفية، مع استبعاد المقارنات البصرية |
| `npm run test:a11y` | اختبار إمكانية الوصول منفردًا |
| `npm run test:visual` | المقارنة البصرية لسطح المكتب والجوال |
| `npm run qa` | بوابة الجودة المحلية الكاملة غير المدمرة |

`npm run qa` لا ينفذ deploy أو migrations أو Vectorize upsert ولا يحتاج إلى أسرار. بعض اختبارات E2E تتطلب أن يكون `local-server.cjs` شغالًا على `127.0.0.1:4173`.

```text

├── migrations/                    # migrations مجهزة لـD1
```

لخريطة أكثر تفصيلًا راجع `docs/project-map.md`.

## الخدمات الخارجية

- Cloudflare Worker وStatic Assets وWorkers AI وVectorize وD1 لمسارات الخادم.
- Google Fonts وCDNJS لبعض الخطوط والأيقونات في الواجهة.
- روابط رسمية وخدمات تعليمية خارجية تفتح في تبويب جديد.
- تكامل Firebase محدود بعداد الزيارات في صفحة أرشيفية.

تفاصيل نطاق الاستخدام والملاحظات التشغيلية موجودة في `docs/external-services.md`.

## النشر

النشر الحالي يعتمد على إعداد Cloudflare Worker وStatic Assets الموجود في المشروع. بصورة عامة:

1. شغّل `npm run qa` وراجع `git diff --check`.
2. راجع التغييرات وتأكد من عدم وجود ملفات بيئة أو أسرار أو مخرجات محلية.
3. تحقق من توفر bindings والخدمات المطلوبة في بيئة Cloudflare دون طباعة قيمها.
4. نفذ أمر النشر المصرح به فقط بعد مراجعة مستقلة وموافقة صريحة.
5. اختبر الصفحات العامة وواجهات API والروابط بعد النشر.

لا تُطبق migrations أو ترفع المعرفة إلى Vectorize ضمن النشر العام تلقائيًا؛ لكل عملية مراجعتها وموافقتها المستقلة.
