# قائمة فحص الجوال والأجهزة اللوحية

هذه قائمة فحص يدوية للنسخة التجريبية. لا تعني إضافة البند هنا أنه اختُبر؛ تُسجّل النتيجة وقت الاختبار الفعلي على المتصفح أو الجهاز.

## المقاسات المطلوبة

- [ ] هاتف 390×844.
- [ ] هاتف 414×896.
- [ ] جهاز لوحي 768×1024.

## خطوات الفحص لكل مقاس

### الصفحة الرئيسية

- [ ] تفتح الصفحة دون أخطاء ظاهرة.
- [ ] يظهر الهيدر والشعار واسم المدرسة بصورة صحيحة.
- [ ] تفتح القائمة الجانبية وتغلق دون تداخل.
- [ ] لا يوجد تمرير أفقي.
- [ ] الأزرار والبطاقات مناسبة للمس ولا تتراكب.

### التسجيل

- [ ] تظهر جميع الحقول ضمن عرض الشاشة.
- [ ] يعمل حفظ بيانات المدرسة.
- [ ] يعمل الدخول كزائر.
- [ ] تبقى القيم المحفوظة ظاهرة عند العودة إلى الصفحة.

### المنصات وركن الدعم

- [ ] يفتح قسم المنصات وتلتف البطاقات دون تداخل.
- [ ] تفتح الروابط الخارجية في تبويب جديد.
- [ ] يظهر ركن الدعم المدرسي وبطاقاته بوضوح.

### مكتبة تقارير المدير

- [ ] تفتح المكتبة وتغلق بسلاسة.
- [ ] يعمل البحث والفلاتر دون تكرار البطاقات.
- [ ] يفتح تخصيص التقرير فوق نافذة المكتبة.
- [ ] يغلق تخصيص التقرير دون بقاء عناصر أسفل الصفحة.
- [ ] يعمل تنزيل Word من زر التقرير.

### مساعد المنصة

- [ ] يفتح المساعد ويغلق دون تغطية عناصر التحكم المهمة.
- [ ] يعمل إرسال سؤال من الحقل ومن الاقتراحات.
- [ ] يظهر الرد ويمكن تمرير المحادثة داخليًا.
- [ ] لا يظهر تمرير أفقي داخل المساعد.

### الفوتر والمظهر

- [ ] يظهر الفوتر دون قص أو تداخل.
- [ ] يعمل الوضع الفاتح وتبقى النصوص واضحة.
- [ ] يعمل الوضع الداكن وتبقى النصوص واضحة.
- [ ] لا تظهر أخطاء حمراء مرتبطة بالواجهة في Console.

## توثيق النتيجة

لكل مقاس، سجّل تاريخ الاختبار، المتصفح أو الجهاز، والنتيجة، وأرفق صورة للمشكلة إن وجدت. تظل البنود أعلاه غير معتمدة حتى تنفيذ الفحص اليدوي الفعلي.
