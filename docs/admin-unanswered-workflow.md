# سير عمل مراجعة الأسئلة غير المجابة

## المراجعة

1. افتح صفحة إدارة الأسئلة غير المجابة وسجّل الدخول بالطريقة المعتمدة.
2. استخدم الحالة والسبب والبحث لتحديد الأسئلة ذات الأولوية.
3. راجع عدد التكرار وصياغة السؤال قبل اتخاذ القرار.
4. تحقق هل السؤال داخل نطاق المنصة والتعليم وهل توجد له إجابة موثقة.

## قرار الإضافة إلى المعرفة

- أضف السؤال عندما يكون متكررًا أو مفيدًا، وتوجد إجابة واضحة في مصدر معتمد.
- لا تخمّن إجابة ولا تعتمد على معرفة عامة غير موثقة.
- ارفض أو تجاهل الأسئلة الخارجة عن النطاق، والمكررة بلا قيمة إضافية، والناقصة التي لا يمكن فهمها بأمان.
- لا تحذف إلا عند وجود سبب واضح وبعد التأكد؛ الحذف لا يمكن التراجع عنه من الواجهة.

## التحويل إلى Markdown

استخدم زر نسخ قالب المعرفة، ثم أكمل القالب داخل الملف المناسب:

```markdown
### نص السؤال؟

الإجابة المؤكدة دون اختصار مخل.

المصدر: اسم المرجع المعتمد.
```

## دورة النشر لاحقًا

تتم الإضافة داخل `knowledge-files/approved`، ثم يُشغّل الفحص المحلي، وبعد المراجعة الصريحة يمكن تنفيذ export ثم upsert. يُختبر الموقع قبل إضافة ملفات Git المحددة وإجراء commit وpush بموافقة صريحة. لا تستخدم `git add .`.
