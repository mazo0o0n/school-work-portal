# معيار ملفات المعرفة

يُستخدم هذا المعيار عند إنشاء أي ملف جديد داخل `knowledge-files/approved` أو مراجعة ملف قائم.

## بيانات الملف

يجب أن يوضح الملف، متى توفرت البيانات:

- العنوان.
- الجهة المصدرة.
- تاريخ الإصدار.
- تاريخ آخر مراجعة بشرية.
- الرابط أو اسم المصدر المعتمد.
- نطاق الصلاحية والفئة المستهدفة.
- ملاحظات الصلاحية أو تاريخ انتهاء التعميم.
- تحذير واضح عندما تكون المعلومات قابلة للتغير.

## تنسيق الأسئلة والأجوبة

كل سؤال يكون عنوان Markdown من المستوى الثالث:

```markdown
### نص السؤال

**الإجابة:**
الإجابة الموثقة والواضحة.

**المصدر:**
اسم الوثيقة أو الرابط المعتمد.

**ملاحظات:**
أي قيد زمني أو تنبيه يلزم لفهم الإجابة.
```

يمكن استخدام عناوين المستوى الثاني `##` لتقسيم الموضوعات، ولا تستخدم للأسئلة لأن عداد المعرفة يعتمد على `###`.

## قواعد الاعتماد

- لا تضف إجابات غير موثقة أو مبنية على التخمين.
- لا تخلط تعميمًا قديمًا بتعميم أحدث دون توضيح النسخ والصلاحية.
- لا تنقل ملفًا إلى `approved` إلا بعد مراجعة بشرية للمحتوى والمصدر.
- لا تضف ملفات `extracted` أو PDF/Word الخام إلى Git.
- بعد أي تعديل معرفة شغّل:

  ```powershell
  node tools/check-knowledge.js
  ```

- لا تنفذ `export-vectors` أو Vectorize upsert إلا يدويًا وبعد موافقة صريحة ومراجعة نتيجة preview.
